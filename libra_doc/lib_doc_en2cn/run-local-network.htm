运行本地网络

libra_swarm 用于在计算机上启动由验证器节点和本地区块链组成的本地网络。 运行本地网络使测试和调试代码更容易得去改变。要查看本地节点生成的日志，请使用 如何访问日志的说明? 你可以使用CLI命令 dev 来编译、发布和执行本地节点集群上的移动中间表示(IR)程序。 有关 本地运行Move程序 请参考更多详情.

libra_swarm 包位于libra存储库的根目录中。

用法

libra_swarm [FLAGS] [OPTIONS]

标记:

  • -l | --enable_logging — Enables logging.
  • -h | --help — Prints help information.
  • -s | --start_client — Starts Libra CLI client.
  • -v | --version — Prints version information.

选项:

  • -n | --num_nodes <num_nodes> — Number of local nodes to start (1 is the default).
  • -c | --config_dir <config_dir> — The directory used by libra_swarm to save the config files, logs, libradb, etc. for the nodes. The user can inspect the information saved in the <config_dir> even after libra_swarm exits. If this directory is unspecified, a temporary directory is used to save this information. The files in <config_dir> are automatically deleted when libra_swarm exits.

先决条件

要安装所有必需的依赖项, 请按所述运行构建脚本 克隆并建立 Libra Core。

运行本地网络

Note: 验证器节点的本地网络将不连接到测试网，目前无法将本地验证器网络连接到测试网.

使用一个或多个验证器节点运行网络并创建本地区块链, 运行 libra_swarm 使用 -n 选项, 如下所示。 如果没有指定-n选项, libra_swarm 将生成一个只有一个节点的网络。

下面的例子启动了一个有4个节点的网络:

$ cd libra
$ cargo run -p libra_swarm -- -s -n 4

该 cargo run -p libra_swarm -- -s 命令执行以下操作:

  • 在计算机上本地生成验证器节点网络。
  • 执行验证器网络的配置管理。
  • 启动Libra CLI客户端实例;客户端连接到本地网络。

验证器网络的配置管理生成:

  • 创世纪事务。
  • Mint密钥——允许执行Mint操作账户的密钥。
  • 本地网络中每个验证器的引导配置——监听端口、系统限制等。

运行cargo命令可能需要一些时间。 如果您已经使用-s选项启动libra_swarm 在成功执行该命令后， Libra CLI客户机的实例将在您的系统上运行。您将看到CLI客户端菜单和 libra% 提示符. 客户端的这个实例连接到您刚才生成的节点的本地网络。现在可以输入CLI命令与本地网络进行交互。

在一个独立的进程中运行Libra CLI客户端 libra_swarm

在前一节中，我们使用-s 选项运行libra_swarm。 这将自动启动Libra CLI客户端和libra_swarm在同一进程中。如果你想在libra_swarm的独立进程中运行CLI客户端，请使用以下步骤:

步骤1：执行 libra_swarm

运行 libra_swarm 不带 -s 选项 如下图所示:

$ cd libra
$ cargo run -p libra_swarm

您将在输出中看到以下信息。

 To run the Libra CLI client in a separate process and connect to the local cluster of nodes you just spawned, use this command:

 cargo run --bin client -- -a localhost -p 57149 -s "/var/folders/xd/sfg4x6713w350lq73kgfc7qxnq5swl/T/.tmpmSSKk9/trusted_peers.config.toml" -m "/var/folders/xd/sfg4x6713w350lq73kgfc7qxnq5swl/T/keypair.ATvJWTliQf0a/temp_faucet_keys"

步骤2：启动CLI客户端的实例

要启动CLI客户端的一个实例并连接到你在步骤1中产生的本地网络:

  • 在新的终端窗口中，转到libra目录。
  • 运行在步骤1的输出中看到的整个命令，如下所示：
$ cd libra
$ cargo run --bin client -- -a localhost -p 57149 -s "/var/folders/xd/sfg4x6713w350lq73kgfc7qxnq5swl/T/.tmpmSSKk9/trusted_peers.config.toml" -m "/var/folders/xd/sfg4x6713w350lq73kgfc7qxnq5swl/T/keypair.ATvJWTliQf0a/temp_faucet_keys"

这将在单独的过程中生成CLI客户端的实例, 您将看到 libra% 提示。

故障排除常见问题

要查看的使用选项 libra_swarm, 请运行：

$ cargo run -p libra_swarm -- -h

如何启用日志记录？

默认情况下，禁用日志记录 libra_swarm. 如果要启用日志记录，运行 libra_swarm 使用 -l 如下所示的选项运行：

$ cargo run -p libra_swarm -- -s -l

如何访问日志？

当你运行cargo run -p libra_swarm -- -s -l命令时，您将在输出中看到以下内容：

Base directory containing logs and configs: Temporary(TempDir { path:"/var/folders/tq/8gxrrmhx16376zxd5r4h9hhn_x1zq3/T/.tmpzeOPC0"})

在另一个终端窗口中，可以使用以下 tail 命令访问日志：

$ tail -f <generated_path>/logs/*

这个 <generated_path> 在上面的例子中是:

/var/folders/tq/8gxrrmhx16376zxd5r4h9hhn_x1zq3/T/.tmpzeOPC0

如何保存日志？

本地验证器节点使用一个临时目录来生成日志和数据库。libra_swarm 进程退出时，将删除此临时目录。要在终止后保留日志 libra_swarm, 运行 libra_swarm 使用中指定的 --c 选项运行 libra_swarm usage.
